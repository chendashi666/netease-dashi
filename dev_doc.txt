2. Download Pipeline (Priority Order)
================================================================================

  1. NetEase old API (/api/song/enhance/player/url) - FREE songs -> FLAC/MP3
  2. NetEase old API + Cookie (MUSIC_U)             - VIP songs  -> FLAC/MP3
  3. yuafeng.cn multi-platform (kw/kg/wy/tx)        - FALLBACK   -> M4A (128kbps)
  4. znnu.com proxy                                  - DOWN (500 upstream failure)

  If all above fail, download_song() fallback calls:
    music_sources.download_with_fallback(song_name, artist, ...)
    -> tries preferred source, then other enabled sources
    -> all routed through yuafeng_api.download_song_by_name()

  After download: Content-Type header is checked and file extension is corrected
  if needed (e.g., server returns audio/flac but filename had .mp3).

4. netease_weapi.py - Weapi Encryption + Cookie
================================================================================

  [Weapi Algorithm]
    1. Generate random 16-byte AES key
    2. JSON params -> AES-128-CBC encrypt (IV="0102030405060708", PKCS7 pad)
    3. Base64 encode -> "params" field
    4. RSA encrypt AES key -> "encSecKey" field (hex, reversed)
    - RSA: 1024-bit public key, exponent e=0x010001 (65537)
    - Key extracted from NetEase PEM certificate

  [Cookie Management]
    set_cookie(str, save=True)  - line 64   - Set and optionally persist
    clear_cookie()              - line 86   - Clear from memory+file
    has_cookie() -> bool        - line 95   - Check if cookie is set
    is_vip() -> Optional[bool]  - line 100  - Test VIP status
    Cookie file: netease_cookie.txt (auto-loaded on module import)

  [VIP Detection] (is_vip(), line 100)
    - Tests with known VIP song (id=1357375695, Beyond - Boundless Oceans)
    - Requests old API download URL with cookie
    - If URL returned -> VIP active (True)
    - If 403/empty -> VIP expired or not VIP (False)
    - If network error -> None (unknown)

  IMPORTANT: weapi endpoint (/weapi/song/enhance/player/url/v1) returns empty
  responses even with valid encryption. Cookie-based VIP auth uses the OLD API
  endpoint (/api/song/enhance/player/url) with MUSIC_U cookie in headers.
  This works correctly for VIP FLAC downloads.

7. ncm_decryptor.py - NCM File Decryption
================================================================================

  [NCM Format]
    - NetEase encrypted music file (.ncm extension)
    - Header: 10 bytes + DWORD key_len + DWORD meta_len + key_data + meta_data
    - After meta: 5 bytes + album_image + actual audio data

  [Decryption Steps]
    1. Read key_data (RC4 encrypted AES key)
    2. RC4 decrypt key_data with CORE_KEY (=687A4852416D736F356B496E62617857)
    3. PKCS7 unpad, slice [17:] -> AES key
    4. Build static RC4 S-box (NOT standard RC4 - S-box unchanged after init)
    5. meta_data XOR with 0x63
    6. Parse: first 22 bytes identifier + base64 encoded metadata
    7. AES-128-ECB decrypt with META_KEY (=2331346C6A6B5F215C5D2630553C2728)
    8. Audio: static keystream RC4 (starts from index 1 in keystream)

  [Common Bugs]
    - Wrong CORE_KEY -> decrypt failure (wrong audio output)
    - Standard RC4 (self-modifying S-box) -> corrupted audio
    - Artist field format: [["name", intID], ...]

  [Usage]
    python ncm_decryptor.py <file|folder> [-o output_dir]

12. Known Issues & Troubleshooting
================================================================================

  [FLAC Availability]
    - Free songs:        NetEase old API -> FLAC (standard/320k/FLAC)
    - VIP + valid Cookie: NetEase old API + MUSIC_U -> FLAC
    - VIP expired cookie: Falls back to standard quality MP3 or yuafeng M4A
    - yuafeng.cn:        Always QQ Music C400 M4A (128kbps), NO FLAC

  [Down / Dead APIs - Do not try to revive]
    - music.znnu.com proxy:        upstream connection failed (500)
    - music.haitangw.cc/kg.php:    code=401 "has not any level"
    - 175.27.166.236/wy.php:       code=201, cookie expired

  [Encoding Issues]
    - All .py files MUST be UTF-8 without BOM (Byte Order Mark)
    - BOM (EF BB BF) causes Python import failures
    - PowerShell "Out-File -Encoding UTF8" adds BOM by default
    - Use: [System.Text.UTF8Encoding]::new($false) to write without BOM
    - Or use: Set-Content with -Encoding UTF8NoBOM
    - Or use: Python open(file, 'w', encoding='utf-8') which does NOT add BOM

  [Python f-string Limitations]
    - Cannot use same quote type inside f-string expression (Python < 3.12)
      Bad:  f"{name["key"]}"       Good: f'{name["key"]}'
    - Cannot contain backslash in expression
      Bad:  f"{name\n}"           Good: use intermediate variable

  [Playlist Limits]
    - v6 API (/api/v6/playlist/detail) returns ALL trackIds (no limit)
    - Batch fetch /api/song/detail?ids=[...] handles up to 100 per request
    - Standard API (/api/playlist/detail) limits to ~10 tracks
    - Confirmed working: 54-track playlist (id=9426599671) parses all tracks
